# EPIC-007: Monetization

Status: Proposed

Primary phase: Phase 4, Business Platform

## Mission

Create sustainable revenue without weakening user trust.

## User Outcome

Users get better planning outcomes, businesses get measurable demand, and NomNomGo can fund ongoing development.

## Stories

| ID | Story | Priority | Acceptance Signal |
| --- | --- | --- | --- |
| MN-001 | As a business owner, I want referral attribution so I know when NomNomGo influenced visits. | Later | Venue clicks, locked plans, map opens, and redemption events can be attributed. |
| MN-002 | As a venue partner, I want partnerships so I can reach groups with planning intent. | Later | Partner venues can be configured and measured without hidden placement. |
| MN-003 | As a power user, I want premium planning so complex plans have better tools. | Later | Premium features are defined without blocking the free core planning loop. |
| MN-004 | As a recurring group, I want premium groups so we can plan more efficiently. | Later | Groups can access advanced history, preferences, or coordination features. |
| MN-005 | As a venue, I want sponsored recommendations so I can promote relevant offers. | Later | Sponsored results are labeled and governed by relevance constraints. |
| MN-006 | As a user, I want event packages so larger outings can be planned with fewer steps. | Later | Packages can bundle venue, activity, and reservation handoff details. |
| MN-007 | As an operator, I want revenue reporting so monetization can be evaluated. | Later | Revenue, conversions, refunds, and partner performance are tracked. |

## Technical Backlog

- Attribution event taxonomy.
- Partner venue model.
- Sponsored result labeling.
- Offer and promotion model.
- Redemption or referral code model.
- Billing architecture proposal.
- Revenue reporting dashboard.
- Trust and transparency guidelines for paid placement.

## Definition Of Done

- Monetization experiments are measurable before they are scaled.
- Paid placement is visible to users.
- Revenue features do not require permanent Differance Labs infrastructure.
- No paid service is introduced without approval.

## Metrics

- Referral conversion rate.
- Partner venue retention.
- Revenue per active group.
- Sponsored result engagement.
- User trust feedback.
- Repeat use after monetized recommendation exposure.
